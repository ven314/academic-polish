# 数学建模竞赛 — AI 时代全流程辅助

## 核心认知

### 什么是数学建模
把现实问题翻译成数学语言 → 用数学工具求解 → 把结果翻译回现实结论。

### 竞赛基本信息（国赛 CUMCM）
- 时间：每年 9 月第二个周四 18:00 至周日 20:00，共 74 小时
- 队伍：3 人一队（编程手 + 建模手 + 写作手）
- 选题：A 题（物理机理）/ B 题（优化决策）/ C 题（大数据）
- 提交：一篇 PDF 论文（摘要 + 正文 + 参考文献 + 附录），无代码要求
- 奖项：国一（前 1%）、国二（前 5%）、省一（前 15%）、省二、省三

### AI 时代的数模新范式
- 代码交给 AI 辅助，团队的绝对重心放在论文上
- 评委看一篇论文平均不超过 10 分钟，决定命运的往往是摘要和前两页图表
- 表面功夫就是硬实力：摘要信息密度、流程图逻辑性、表格美观度

## 模型选择与实施体系

### 第一类：评价类模型（选谁最好 / 排序 / 评级）

| 模型 | 适用场景 | 数据要求 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **层次分析法（AHP）** | 指标少（≤15）、主观判断 | 专家打分 | 简单直观 | 主观性强、指标多时一致性差 |
| **熵权法** | 指标多、数据完整 | 面板数据 | 客观、自动定权 | 无法处理定性指标 |
| **TOPSIS** | 多方案择优 | 决策矩阵 | 原理简单、计算快 | 对权重敏感 |
| **灰色关联分析** | 小样本 / 数据少 | 数列 $n \geq 3$ | 适合小样本 | 分辨率参数影响大 |
| **模糊综合评价** | 评价边界模糊 | 隶属度矩阵 | 处理不确定性 | 隶属函数难以确定 |
| **CRITIC** | 指标含冲突性 | 面板数据 | 同时考虑对比度和冲突性 | 比熵权法更复杂 |

**评价类选题标志：** 问题中出现"评价""排名""优劣""综合效益""绩效评估"

**优先策略：** 客观数据充足 → 熵权法或 CRITIC；主观因素多 → AHP；小样本 → 灰色关联

### 第二类：预测类模型（未来怎样 / 趋势）

| 模型 | 适用场景 | 数据要求 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **多元线性回归（MLR）** | 连续值预测，线性关系 | 30+ 样本，无多重共线性 | 可解释性强、简单 | 不能处理非线性 |
| **指数平滑（ETS）** | 单变量、有趋势/季节 | 平稳/趋势序列 | 参数少、适合短期预测 | 不能多变量 |
| **ARIMA** | 单变量时间序列 | 50+ 时间点、平稳 | 经典框架成熟 | 不能处理非线性 |
| **灰色预测 GM(1,1)** | 小样本（4-8 个数据） | 指数增长型数据 | 小样本优势 | 仅限单变量齐次指数 |
| **LSTM** | 多变量时序、长期依赖 | 大量数据（1000+ 时间步） | 捕捉长期依赖 | 参数多、需要 GPU |
| **XGBoost / LightGBM** | 多特征、非线性、分类/回归 | 结构化数据 | 精度高、特征重要性 | 调参复杂 |

**预测类选题标志：** "预测""趋势分析""预估""预报""估算"

**优先策略：** 时间序列数据充足 → ARIMA；小样本 → 灰色预测；多特征分类 → XGBoost；多变量长序列 → LSTM

### 第三类：优化类模型（怎么最好 / 资源配置）

| 模型 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **线性规划** | 目标+约束均为线性 | 解的存在性保证 | 局限性大 |
| **整数规划** | 决策变量为整数 | 适用离散决策 | NP-hard |
| **多目标规划** | 多个冲突目标 | 贴近实际 | 权重难定 |
| **动态规划** | 多阶段决策 | 最优子结构 | 维数灾难 |
| **粒子群算法（PSO）** | 非线性 / 不连续 | 不需求导 | 可能早熟 |
| **遗传算法（GA）** | 组合优化 | 全局搜索强 | 收敛慢 |
| **模拟退火** | 大规模组合优化 | 理论上收敛到全局最优 | 调参敏感 |

**优化类选题标志：** "最优""最小化""调度""分配""路径规划""策略"

**优先策略：** 线性目标→线性规划；整数变量→整数规划；多目标→NSGA-II 或分层求解；非线性复杂问题→PSO/GA

### 第四类：分类与聚类模型（分几类 / 离群检测）

| 模型 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **K-Means** | 大样本、球形簇 | 计算快、简单 | 需指定 K、不能非球形 |
| **DBSCAN** | 任意形状簇 | 自动判 K、抗噪 | 密度不均匀时效果差 |
| **层次聚类** | 小样本 | 结果可视、树状图 | 计算量大 |
| **PCA** | 降维、去噪 | 降维可视化 | 损失可解释性 |
| **t-SNE** | 可视化高维数据 | 保留局部结构 | 不可逆、不适用于预测 |

**分类/聚类选题标志：** "分类""聚类""降维""类型识别""异常检测"

## 数据预处理

### 缺失值处理
| 方法 | 适用条件 | 代码 |
|------|---------|------|
| 删除含缺失的行 | 缺失率低（<5%） | `df.dropna()` |
| 均值/中位数填充 | 数值型、随机缺失 | `df.fillna(df.mean())` |
| 众数填充 | 分类型 | `df.fillna(df.mode()[0])` |
| 前向/后向填充 | 时间序列 | `df.fillna(method='ffill')` |
| 插值法 | 有趋势的序列 | `df.interpolate()` |
| KNN 填充 | 特征间有相关性 | `from sklearn.impute import KNNImputer` |

### 异常值处理（三西格玛法 + IQR 法）
```python
# 3σ 法：超出 μ±3σ 视为异常
df[(df['col'] >= df['col'].mean() - 3*df['col'].std()) & 
   (df['col'] <= df['col'].mean() + 3*df['col'].std())]

# IQR 法：超出 Q1-1.5×IQR 或 Q3+1.5×IQR 视为异常
Q1, Q3 = df['col'].quantile(0.25), df['col'].quantile(0.75)
IQR = Q3 - Q1
df[(df['col'] >= Q1 - 1.5*IQR) & (df['col'] <= Q3 + 1.5*IQR)]
```

### 数据标准化
| 方法 | 公式 | 适用 |
|------|------|------|
| Z-score | $x' = (x-\mu)/\sigma$ | 无界数据，SVM/PCA/K-Means |
| MinMax | $x' = (x-x_{\min})/(x_{\max}-x_{\min})$ | 有界数据，神经网络 |
| Robust | $x' = (x-\text{median})/\text{IQR}$ | 含异常值的列 |

## AI 辅助建模

### AI 生成代码的基本原则
1. AI 生成代码必须验证，不能直接使用
2. 将复杂问题拆解为几步分别问 AI
3. 提供上下文：数据样例、预期输出、约束条件
4. AI 代码出错时使用万能调试模板

### 论文插图规范（国赛评审标准）
- **摘要插图**：必须包含！流程图（方法）+ 结果图（效果）
- **正文插图**：数据类题目至少 6 张图，其中 1-2 张高级可视化
- **表格规范**：三线表（toprule/midrule/bottomrule）
- **避免**：3D 饼图、颜色过多、截图代替绘图的图片
- **配色**：红绿蓝黑四色足够，不用渐变色、不用色块填充

### 常用高级可视化选型
| 目的 | 图表类型 |
|------|---------|
| 指标权重展示 | 雷达图（Radar Chart） |
| 特征筛选 | 热力图（Heatmap）+ 相关系数矩阵 |
| 多方法对比 | 柱状图 + 误差线 |
| 模型性能 | ROC 曲线 / P-R 曲线 |
| 降维展示 | PCA / t-SNE 散点图 |
| 空间分布 | GIS 热力图 / 散点分层 |
| 趋势对比 | 折线图 + 置信区间 |
| 分类结果 | 混淆矩阵热力图 |

### LaTeX 论文排版要点
1. 使用 ctexart 文档类（中文）
2. 公式必须用 `\begin{equation}`，编号连续
3. 图表必须引用（"如图 X 所示""如表 X 所示"）
4. 参考文献使用 BibTeX 或 thebibliography
5. 摘要控制在半页以内，包含：问题+方法+结果（有数值）+结论
6. 三线表代码格式：
```latex
\begin{tabular}{lccc}
\toprule
模型 & 准确率 & 召回率 & F1 \\
\midrule
Method A & 0.95 & 0.93 & 0.94 \\
\bottomrule
\end{tabular}
```

### 团队分工（推荐）
- **编程手**（全程 AI 辅助）：负责所有代码 + AI 协调
- **写作手**：第一天就开始写论文骨架，数据来一段更新一段

### 论文评判标准（评委视角）
1. 摘要（30%）：问题+方法+数值结果+结论，四要素缺一不可
2. 模型合理性（25%）：方法选择论证、假设合理性
3. 结果分析（20%）：图表规范性、结果讨论深度
4. 创新性（15%）：多方法对比、灵敏度分析、模型改进
5. 排版质量（10%）：公式规范、表格美观、参考文献完整
